## 项目背景

用户希望构建一个 **Web 游戏平台**，用于：
1. 在平台上开发各种小游戏
2. 提升后端 + 分布式能力
3. 作为简历项目

---

## 项目需求

| 维度 | 需求 |
|------|------|
| **核心功能** | 游戏大厅、实时多人游戏、好友系统 |
| **实时通信** | 需要 WebSocket 支持实时联机 |
| **技术栈** | Java/Spring Boot（后端）、Vue 3（前端） |
| **前端策略** | AI 辅助编写前端代码，用户负责对接 + 学习 |
| **数据库** | MySQL + Redis |
| **部署** | 阿里云服务器 |
| **时间线** | 1-3 个月上线第一款游戏 |

---

## 用户技术背景

- 后端：熟悉 Spring Boot + Java WebSocket（有做过实时迷宫游戏的经验）
- 前端：刚入门，计划让 AI 帮忙写
- 架构：零基础，希望"一开始就设计正确"

---

## 项目分期规划（草稿）

### Phase 1：基础设施（2-3 周）
- 账号系统（注册/登录/JWT）
- 好友系统（添加/删除/列表）
- WebSocket 基础设施（连接管理/消息路由/心跳）

### Phase 2：第一款游戏（2-4 周）
- 游戏大厅（房间列表/创建/加入）
- 游戏框架（状态同步/帧同步）
- 第一款游戏（贪吃蛇或五子棋）

### Phase 3：完善与扩展（4-8 周）
- 排行榜
- 分布式改造（多节点部署）
- 更多游戏
